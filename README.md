# Employee, Department & Project Management API (.NET 5) - Moraru Andrei Virgil - Internship Technical Test

## What this project is
A REST API built with ASP.NET Core Web API (.NET 5) using:
- SQL Server (database)
- Entity Framework Core (code-first + migrations)
- Swagger (for testing endpoints)
- External API (Codito) to generate project codes

## Requirements to run
- Visual Studio 2022
- .NET 5 SDK
- SQL Server installed (SSMS optional but helpful)

## 1) Configure the database connection
Open `appsettings.json` and set:

ConnectionStrings:Default

Example (local default SQL Server instance):
Server=.;Database=CompanyManagerDb;Trusted_Connection=True;

If your SQL Server is SQLEXPRESS:
Server=localhost\SQLEXPRESS;Database=CompanyManagerDb;Trusted_Connection=True;

## 2) Create the database (EF Core migrations)
### Visual Studio (recommended)
Tools → NuGet Package Manager → Package Manager Console

Run:
Update-Database

This will create the database and tables automatically using the migrations included in this repository.

## 3) Run the API
- Run from Visual Studio: press F5
OR
- Run from terminal in the project folder:
  dotnet run

## 4) Test in Swagger
Open:
https://localhost:<port>/swagger

### Important note about Swagger “Try it out”
Swagger may show extra fields (like `id` or `projectCode`) in the example JSON.  
That is normal when endpoints accept full entity models. You can still test successfully by sending only the fields shown below.

### C) Projects (External API + Bonus requirement)
POST /api/projects
{
  "name": "New Website",
  "budget": 100000
}

When a project is created:
1) A project row is inserted into the database (to generate Project Id).
2) A random code is generated by calling the Codito API.
3) The Project Id is appended to the random code: <random>-<projectId>
4) The final code is saved as ProjectCode.

Project creation is done inside a database transaction so if any step fails,
the whole operation is rolled back and no project exists without a ProjectCode.

GET /api/projects
GET /api/projects/{id}
PUT /api/projects/{id}
DELETE /api/projects/{id}
